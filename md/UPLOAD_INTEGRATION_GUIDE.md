# 文档上传功能集成指南

## 📋 问题解决

之前独立的 `/upload/` 页面与现有的前端系统冲突，导致页面无法正常启动。现已将文档上传功能**集成为管理员仪表板中的一个模态框窗口**。

## 🎯 集成方案

### 前端集成
- **位置**：在 `web_app/templates/admin_dashboard.html` 的"管理快捷操作"中添加新按钮
- **样式**：完全采用现有设计系统（蓝绿渐变主题）
- **交互**：点击"📤 文档上传"按钮即可打开模态框

### 后端修改
- **独立页面路由**：`GET /upload/` 现已改为返回信息提示，不再渲染 HTML 页面
- **API 端点保留**：
  - `POST /upload/preview` - 上传并预览文件解析结果
  - `POST /upload/confirm` - 确认数据并导入数据库

## 🚀 使用方式

### 1. 启动应用
```bash
cd e:\health_bigdata_system
# 根据你的启动方式运行 Flask 应用
python run.py
# 或使用 docker-compose
docker-compose up
```

### 2. 访问管理员仪表板
```
http://localhost:5000/admin/dashboard
```

### 3. 使用文档上传
1. 在"管理快捷操作"区域找到"📤 文档上传"按钮
2. 点击打开模态框窗口
3. 拖拽或点击选择文件（支持 Excel、PDF、Word、TXT）
4. 系统自动识别数据类型和目标表
5. 预览数据后点击"确认入库"导入数据库

## 📊 支持的数据类型

系统会自动识别上传文件的内容并映射到合适的表：

| 关键词 | 目标表 | 说明 |
|------|------|------|
| 人口、population | population_info | 人口数据 |
| 机构、hospital、医院 | medical_institution | 医疗机构数据 |

## 🔧 故障排除

### 问题：模态框不显示
**解决**：确保在浏览器开发者工具中检查是否有 JavaScript 错误

### 问题：文件上传失败
**解决**：
1. 确保 `web_app/uploads/` 文件夹存在且有写权限
2. 检查文件大小是否超过 100MB
3. 查看浏览器控制台中的具体错误信息

### 问题：数据入库失败
**解决**：
1. 确保 MySQL 数据库正在运行且配置正确 (`localhost:3307`)
2. 确保目标表存在于 `health_db` 数据库中
3. 查看Flask 应用日志中的错误信息
